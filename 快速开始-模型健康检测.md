## 📍 如何使用

### 第一步: 启动应用

开发服务器已经在运行:
- 本地访问: http://localhost:3001/
- 网络访问: http://192.168.1.4:3001/

如果未启动,运行:
```bash
cd /Users/sangsang/NovelistSuite
npm run dev
```

### 第二步: 访问健康检测器

1. 打开浏览器访问 http://localhost:3001/
2. 点击"管理员登录"
3. 切换到"模型配置"标签
4. 点击右上角的"**测试模型健康**"按钮

### 第三步: 测试模型

**单个测试**:
- 找到想测试的模型
- 点击右侧的"测试"按钮
- 查看延迟和状态

**批量测试**:
- 点击右上角"全部测试"按钮
- 等待所有模型测试完成
- 对比延迟选择最快的模型

## 📊 测试结果说明

### 健康状态图标

- ✅ **绿色勾号**: 模型健康,可正常使用
- ❌ **红色叉号**: 模型异常,检查错误信息
- ⚠️ **黄色警告**: 模型不支持或待实现
- 🔄 **旋转图标**: 正在测试中

### 延迟等级

- 🟢 **优秀** (< 1秒): 响应非常快
- 🔵 **良好** (1-3秒): 响应速度不错
- 🟡 **一般** (3-5秒): 可接受的速度
- 🔴 **较慢** (> 5秒): 网络可能有问题

## ⚙️ 配置说明

### API Key 配置

确保 `.env` 文件中配置了正确的 API Key:

```env
GEMINI_API_KEY=你的API密钥
```

### 模型配额

在"模型配置"标签中可以查看和修改:
- **RPM** (每分钟请求数)
- **每日配额** (每天请求数)
- **上下文窗口** (Token 数量)

## 🔍 故障排查

### 问题: 所有模型都显示"API密钥无效"

**解决方案**:
1. 检查 `.env` 文件是否存在
2. 确认 `GEMINI_API_KEY` 是否正确
3. 重启开发服务器: `npm run dev`

### 问题: 千问/豆包模型显示"功能待实现"

**说明**: 这是正常的! 当前版本主要支持 Gemini 系列。千问和豆包的完整支持将在后续版本添加。

### 问题: 延迟很高 (> 5秒)

**可能原因**:
- 网络连接不稳定
- 模型服务器负载高
- 需要配置代理

**建议**:
- 尝试其他模型
- 检查网络连接
- 稍后重试
